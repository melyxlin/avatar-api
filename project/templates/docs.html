Documentation

Introduction

This is the documentation for Version 1 (V1) of the Avatar API, originally released in September of 2019

Reponse Format

The default response format is JSON

Authentication

This is an open API, which means that no authentication is required to query the API for data. Therefore, there is only support for GET-ing data.

ROUTES

All routes must be prefixed with https://www.avatarapi/herokuapp.com

Characters Route
GET /characters

Returns all characters.

Name Type Description
name string Name of character
gender string Gender of character
ethnicity string Ethnicity of character
nationality string Nationality of character
bending array of strings The bendings that the character can do. If character is a nonbender or undisclosed, then 'Nonbending'
year of birth string Character's date of birth
year of death string Character's date of death
roles array of strings Characters's role during their lifetime
aliases array of strings Aliases that the character has used
affiliations array of strings Affiliations that the character belongs to
weapons array of strings Weapons that the character uses
fighting styles array of strings Fighting styles that the character uses

Example response:

      "name": "Aang",

      ...

Filtering Characters

You can also include filter parameters in your request to the characters endpoint by the following url:

GET /characters/param/{value}
Param Value Description
benders string Characters with the given bending skills are included in the response.
gender string Characters with the given gender are included in the response.
ethnicity string Characters with the given ethnicity are included in the response.
nationality string Characters with the given nationality are included in the response.
affiliation string Characters who are part of the given affiliation are included in the response.

Locations Route
GET /locations

Returns all the locations.

Name Type Description
name string Name of location
nation string Nation where location is located
location string General location of where in the nation the location is located
role string The main function of the location

Example response: 


  [
    {
      "name": "Eastern Air Temple",
      "nation": "Air Nomad Mountain Range",
      "location": "Southeast of Earth Kingsdom",
      "role": "Home to Female Air Acolytes and Air Nomads"
    },
    {
      "name": "Ba Sing Se",
      "nation": "Earth Kingdom",
      "location": "Northeastern Earth Kingdom",
      "role": "Capital of the Earth Kingdom"
    },
    ....
  ]

Filtering Locations

You can also include filter parameters in your request to the locations endpoint by the following url:

GET /locations/param/{value}
Param Value Description
nations string Locations in the given nation are included in the response.

Getting a specific location

You can also get a specific location by the following url:

GET /locations/{name}

Example response: 


  {
    "name": "Eastern Air Temple",
    "nation": "Air Nomad Mountain Range",
    "location": "Southeast of Earth Kingsdom",
    "role": "Home to Female Air Acolytes and Air Nomads"
  }
